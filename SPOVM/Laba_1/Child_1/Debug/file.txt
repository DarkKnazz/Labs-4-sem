В данной части будет произведено разделение проекта на наиболее общие части(блоки)
и будет проведено описание работы, взаимодействия между собой. 
Также в данной части будут рассмотрены дополнительные модули,
вспомогательные файлы, программы, необходимые для проекта.
Для более наглядного понимания работы программы приведена 
структурная схема, на которой показаны все блоки программы,
а так же их взаимоотношения между собой, т.е. какой блок к какому блоку имеет доступ,
куда передает данные и от кого их получает.
Программу можно разделить на такие блоки, как блок пользователя, блок логики, блок файлов,
блок вычислений, блок вывода на экран. 
На мой взгляд, самый важный блок – это блок логики, т.к. он отвечает за принятие решений или
за реакцию на действия пользователя. Также блок логики взаимодействует со всеми другими блоками,
т.к. блок логики направляет сигналы в тот или иной блок, 
чтобы тот произвел некоторые действия и вернул результат назад.
Также блок логики заполняет данные о билете, 
введенные пользователем и отправляет для дальнейшей работы в блок файлов.
Блок пользователя не менее важен, так как 
именно он подает сигналы о том, что нужно сделать блоку логики.
Пользователь взаимодействует только с блоком логики. 
Пользователь выбирает пункты меню, выведенные на экран и отправляет данные блоку логики. 
Также пользователь вводит данные билета и просматривает бронированные.
Общение между пользователем и блоком логики организовано с помощью потока ввода.
Блок вывода на экран не очень большой по объему и по уровню написания, однако он не менее важен,
так как отвечает за вывод результатов работы программы пользователю. Он выводит на экран все,
начиная с информации о программе, меню выбора до данных о билете и всех бронированных билетов. 
Также пользователь может вывести при желании расписание автобусов на экран. 
Данные для вывода блок получает из блока логики, который в свою очередь, 
получает их от пользователя или из блока файлов. 
Вывод осуществляется с помощью стандартного потока вывода 
И перегруженного оператора вывода в поток для упрощения вывода данных о билете. 
Блок вывода на экран использует псевдографику.
Следующий блок, который будет рассмотрен – блок вычислений. Самый большой по объему кода и сложности блок. 
В блоке вычислений, как следует из названия, происходят всевозможные вычисления и работа с данными. 
В самом коде блок вычислений представлен всевозможными методами классов для обработки данных о билетах. 
В основном блок вычислений получает взятый из блока файлов массив билетов и запрос от пользователя, 
основываясь на котором и выполняет дальнейшие действия, т.е. к примеру найти по номеру телефона нужные билеты, 
занести их в массив и вернуть в блок логики, найти нужный билет, удалить билет, 
данные о котором будут сообщены пользователем. Также через данный блок идет сортировка билетов
по дате(если пользователь захочет). Хочу заметить, что блок вычислений не имеет прямого доступа ни к блоку пользователя,
ни к блоку файлов, несмотря на то, что получает из них данные. Единственный блок, с которым сообщается
блок вычислений – блок логики. От него он получает необходимые данные и указание, что с ними делает,
ему же и отдает результат, а блок логики уже распределяет по назначению.
Последний наиболее общий блок – это блок файлов. Блок файлов – своеобразное хранилище данных. 
Представлены бинарными файлами и средствами для работы с ними. Блок файлов существует на случай аварийного или 
случайного завершения программы, а также для облегчения работы с большим количеством данных о билете. 
Само собой, блок выполняет функцию и хранения тоже. В блоке файлов также хранится файл с расписанием автобусов, 
который пользователь может по цепочке через блок логики вывести на экран и просмотреть. 
Блок файлов непосредственно общается только лишь с блоком логики, 
получает от него данные и заносит в файл, или же достает из файла данные и передает их блоку логики. 
Общение в файловом блоке организовано с помощью файловых потоков ввода\вывода, а также перегруженных потоков 
ввода\вывода для облегчения занесения данных о билетах в файл. 
Как мы можем увидеть, взаимодействие между блоками программы осуществляется при помощи потоков ввода\вывода. 
Только с блоком вычислений существует другой вид взаимоотношений. Блок вычислений – блок состоящий из 
различных функций и методов классов, поэтому общение с ним выполняется при помощи передачи данных в виде 
массивов\векторов\листов по значению или по ссылке(в зависимости от ситуации). Возврат результата осуществляется 
с помощью ключевого слова return.
